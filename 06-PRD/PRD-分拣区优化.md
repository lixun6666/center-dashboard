## 一、背景与目标

**背景**：分拣区是**分拣组长**（也供仓长/调度查看）掌握分拣进度、组织人力的核心页面。现状指标较全，但存在：站点/路区重复、完成类数据卡住、缺质量/效率呈现、展示偏平铺等问题。

**目标**：在现有分拣区基础上做增量优化，使其支撑分拣组长"**看工作量 → 预判来货高峰 → 盯消化/积压 → 控质量/效率 → 复盘**"的完整决策链。

**衡量**：分拣组长可在页面内判断"**来货消化得过来吗、待分拣多不多、要不要增援、质量是否异常**"；关键数据准确可信。

> 注：分拣为**来货即分**模式，分拣组长**不以截单时间为作业节点**；进度＝来货与已拣的差（积压）与趋势，非"距截单倒计时"。

***

## 二、范围

**范围内**：分拣区页面的展示优化、口径修正、交互增强、必要新增指标。
**范围外**：跨页（概况）改造；分拣机/DMS 等外部系统改造；纯离线历史分析（走运营驾驶舱）。

***

## 三、用户与场景

- **主用户**：分拣组长——24h 作业按早/中/晚小班次盯现场，**来货即分**，据数据派人力、定优先流向、控制积压（不盯截单）。
- **次要用户**：仓长（全局/产能对比）、运输组长（看已分拣量配载）。

***

## 四、时间口径（全局前提，本页遵循）

- 基准：**现场操作日时间为大窗口**（按中心配置），页面默认展示当前操作日。
- 可选：在该操作日内，可按**分拣班次（早/中/晚小班次）**筛选查看。
- 不提供任意自定义时间；历史（昨日/过去一周）走运营驾驶舱离线。
- 详见专题1 / 《PRD-时间口径对齐》。

## 五、功能需求

### FR-0 查询 / 筛选栏

- **需求**：工具栏含——中心选择——、操作日（按中心固定大窗口，只读起止）、今日 / 明日切换、集货截单 / 散货截单两个截单时间展示、查询；分拣班次筛选置于页签行、标二期（全天/早/中/晚，依赖班次时间取数）。
- **口径**：见四·时间口径。
- **验收标准**：
  - AC1 操作日窗口随中心变化、只读展示起止；
  - AC2 提供今日/明日切换；
  - AC3 展示集货/散货截单时间；
  - AC4 分拣班次筛选为二期、仅展示标签不可交互；
  - AC5 不提供任意自定义日期。

### FR-1 关键数据卡（KPI · 6 项 · 数字优先）

- **目的**：将核心指标定义为独立指标卡片，**重要数字放大展示**，符合"概览看图、细节看表 + 数字优先于图表"原则（S-2/US-D14）。
- **需求**：
  - 6 项指标卡片：应分拣（二期·占位）、已分拣、待分拣、来货速度、来货消化率、回流率。
  - 应分拣 / 已分拣 / 待分拣 卡片可点击下钻；来货速度 / 来货消化率 / 回流率 无点击效果。
  - 下钻（右侧抽屉）：
    - 应分拣：顶部汇总（应分拣/今日已入库-应分拣/历史留仓-应分拣 三卡）+ 时间分布（合计/＜24H/48H/72H/＞72H）+ 目的服务渠道；
    - 已分拣：顶部汇总（已分拣/袋数/目的组织 三卡）+ 目的流向·分小时明细；
    - 待分拣：顶部汇总（待分拣/今日已入库-待分拣/历史留仓-待分拣 三卡）+ 时间分布 + 目的流向；
  - 时间分布/目的流向中的**运单数**可下钻运单明细（三级 lightbox）。
- **验收标准**：
  - AC1 6 项指标卡片正确展示，重要数字放大突出；
  - AC2 应分拣卡片点击下钻，展示三汇总卡 + 时间分布 + 目的服务渠道；
  - AC3 已分拣卡片点击下钻，展示三汇总卡 + 目的流向·分小时明细；
  - AC4 待分拣卡片点击下钻，展示三汇总卡 + 时间分布 + 目的流向；
  - AC5 来货速度、来货消化率、回流率无点击效果；
  - AC6 时间分布/目的流向中的运单数可下钻运单明细（三级 lightbox，关闭不联动二级抽屉）。

### FR-2 分拣票数趋势（折线图 · 图表区）

- **现状**：早期版本多线累计趋势图，线多偏乱、默认全开看不清重点。
- **需求**：
  - 图表区放折线图，X 轴=操作日时间（按小时排布），Y 轴=数量（票）；支持累计/小时切换（二期占位）。
  - 主图默认 4 线：分拣票数、分拣机分拣、人工分拣、同比上周；
  - 可选 3 线默认隐藏：环比昨日、来货票数、出站票数；点击图例可加显/隐藏。
  - 辅助统计卡：峰值、谷值、均值、当前时段 4 张迷你统计卡。
  - 小时模式下 UI 明确标注「小时完成票数」（调研3.3），避免误解为累计值。
- **口径**：分拣票数=本操作日已分拣运单数（不限入库日），按小时（可累计）；同比=上周同一时段；环比=昨日同一时段。
- **验收标准**：
  - AC1 主图默认显示分拣票数、分拣机分拣、人工分拣、同比上周 4 条线；
  - AC2 环比昨日、来货票数、出站票数默认隐藏、可经图例开启；
  - AC3 辅助统计卡（峰值/谷值/均值/当前）正确展示；
  - AC4 小时模式标注「小时完成票数」（二期）。

**指标逻辑（FR-2）**

> 主图默认 4 线、可选 3 线，统一定义如下。

| 线     | 逻辑/口径                                | 默认       | 状态                  |
| ----- | ------------------------------------- | -------- | ------------------- |
| 分拣票数  | 本操作日内完成分拣动作的运单数（不限入库日），按小时（可累计）   | 默认显示     | ✅                   |
| 分拣机分拣 | 分拣机完成的分拣运单数                          | 默认显示     | ✅                   |
| 人工分拣  | 人工完成的分拣运单数                           | 默认显示     | ✅                   |
| 同比上周  | 当日 vs 上周同一操作日、同一小时段的分拣票数           | 默认显示     | ✅                   |
| 环比昨日  | 当日 vs 昨日同一小时段的分拣票数                  | 可选·默认隐藏  | ❓ 取数               |
| 来货票数  | 按小时新入库运单数                            | 可选·默认隐藏  | ✅                   |

### FR-3 已分拣 × 目的流向（产能 · 下游预报）

- **目的**：供下游预报货量，反映"已分拣产能按目的组织的分布"。
- **现状**：早期粗分（按 Hub/服务渠道）+ 细分（按站点/路区）两张表，按小时列展开。
- **需求**：
  - 整体以"已分拣 · 目的流向"为主题；已分拣同时显示**票数 + 袋数**两个维度（票数反映分拣量、为分拣组长主看指标；袋数供下游报量参考）。
  - 顶部：**目的流向分类标签**（全部 / hub / 站点 / 路区 / 服务渠道），点击切换对应维度数据（二期占位）。
  - 视图切换：表格 | 图表切换按钮，默认表格视图（二期占位）。
  - 表格视图：按目的组织 + 分小时列展示，含合计票数、合计袋数、各小时票数。
  - 图表视图：按目的组织展示分小时折线图（二期）。
  - 单小时 | 累计 切换：表格/图表均支持（二期占位）。
  - 展开/全部 按钮：点击打开已分拣下钻抽屉。
- **口径**：合计票数=已分拣（不限入库日）=运单数；合计袋数=去重袋数（袋=袋/代牌）；干线/支线判定规则待业务确认。
- **验收标准**：
  - AC1 顶部含目的流向分类标签（占位）；
  - AC2 表格视图按目的组织 + 分小时列展示，含合计票/袋/各小时票；
  - AC3 含 CBT 行；
  - AC4 合计袋数为去重值；
  - AC5 展开/全部按钮打开已分拣下钻抽屉。

### FR-4 待分拣（口径修正）

- **现状**：待分拣按流向拆 hub/站点/路区/服务渠道，但站点=路区取同值（P1，多次复现）。
- **需求**：
  - ① 待分拣阶段流向不分站点/路区（此阶段无法区分），合并为"站点/路区"或仅保留可区分维度；hub+站点+服务渠道=待分拣总数关系清晰。
  - ② 待分拣区分结单前/结单后、剔除非本班次货：待分拣量应能区分"结单前应处理"与"结单后到货"，且不含非本班次的货，避免把不该本班消化的货计入积压、误导增援判断。
- **口径**：结单前/后以中心结单时间为界；"本班次货"按所选分拣班次窗口判定。
- **验收标准**：
  - AC1 待分拣流向明细不再出现站点、路区数值相同的重复列；
  - AC2 流向明细加总与总数关系可解释；
  - AC3 待分拣可按结单前/结单后区分展示，且默认剔除非本班次货。

**指标逻辑（FR-4）**

| 维度       | 逻辑/口径                              | 状态                |
| -------- | ---------------------------------- | ----------------- |
| 站点/路区合并 | 待分拣阶段流向无法区分站点与路区，合并展示            | ✅ 修 P1            |
| 结单前/后    | 以中心结单时间为界，结单前/后分列                 | ❓ 取数（Q12）         |
| 剔除非本班次货 | 默认按所选分拣班次窗口过滤，非本班次不计入积压           | ❓ 取数（Q12）         |

### FR-5 分拣机 · 人工 + 分拣机状态

- **需求**：
  - **分拣机·人工构成（饼图）**：展示分拣机与人工分拣的比例，含百分比与票数。
  - **分拣机状态（柱+线）**：按小时呈现分拣量（柱状图）+ 回流率（折线图），双 Y 轴。图例含分拣量和回流率两项，默认均选中。
  - 移除原有统计卡片：今日总拣量、预出步 5·总拣量、回出 5·总拣量、平均流失率、预出 5·平均回流率、回出 5·平均回流率。
- **验收标准**：
  - AC1 饼图正确呈现分拣机/人工构成；
  - AC2 柱+线图正确呈现分拣量与回流率；
  - AC3 图例可切换显隐；
  - AC4 不展示统计卡片。

**指标逻辑（FR-5）**

| 指标         | 逻辑/口径                       | 状态            |
| ---------- | --------------------------- | ------------- |
| 分拣机·人工构成（饼图） | 分拣机分拣票数 / 人工分拣票数，含百分比与票数  | ✅             |
| 分拣量（柱）     | 每小时分拣量（与主指标同口径）            | ✅             |
| 回流率（折线）    | 流失件 ÷ 分拣件，单位 %             | ❓ 取数（Q3）     |

### FR-6 分拣时效（12/18/24h 多节点）

- **需求**：分拣率按 12h / 18h / 24h 多节点展示（各节点：截至该时长的已分拣占应分拣比例），含进度条+百分比+状态标签（正常/临近/超时）。
  - 超时预警颜色：正常绿、临近超时黄、已超时红。
- **口径**：分拣率=对应时长内已分拣运单数 ÷ 应分拣运单数（节点口径以质控为准）。
- **验收标准**：
  - AC1 分拣率按 12/18/24h 多节点呈现、进度条+百分比；
  - AC2 正常绿、临近黄、超时红颜色预警；
  - AC3 口径与质控考核对齐。

**指标逻辑（FR-6）**

| 节点      | 逻辑/口径                          | 状态                |
| ------- | ------------------------------ | ----------------- |
| 12h 分拣率 | 12h 内已分拣运单数 ÷ 应分拣运单数          | ❓ 取数+质控口径（Q11）  |
| 18h 分拣率 | 18h 内已分拣运单数 ÷ 应分拣运单数          | ❓ 取数+质控口径（Q11）  |
| 24h 分拣率 | 24h 内已分拣运单数 ÷ 应分拣运单数          | ❓ 取数+质控口径（Q11）  |
| 颜色预警    | 正常绿、临近（如 20h）黄、已超时红          | ❓ 阈值待质控确认         |

### FR-7 备货链（预计到达 → 已到未卸 → 待分拣）

- **需求**：把"预计到达 → 已到未卸 → 待分拣"串成一条链展示，让分拣组长判断"**此刻多少要分、后面还有多少在赶来**"，据此提前排人力。
  - 布局：三列并排 + 箭头连接。
  - 预计到达与已到未卸节点：票数大字前置、袋数次之（票数反映工作量、袋数供下游报量参考）。
  - 待分拣节点：按票数展示，**高亮突出**（核心）。
  - 预计到达定位为**前瞻参考**（车在路上，还要经卸车/入库），非核心。
- **下钻**（右侧抽屉）：
  - 预计到达：预计到车辆数、袋数、票数 + 目的已到达明细（目的服务渠道/路区/袋/票/占比）；
  - 已到未卸：已到未卸车辆数、袋数、票数 + 滞留时长分布（合计/0-0.5H/1-3H/3-6H/6-12H/>12H，各含车辆/袋数/运单数）；
  - 待分拣：同 FR-1 待分拣下钻。
- **验收标准**：
  - AC1 备货链按"预计到达→已到未卸→待分拣"顺序三列并排呈现；
  - AC2 预计到达标注为前瞻参考；
  - AC3 各节点支持点击下钻；
  - AC4 待分拣节点高亮突出。

**指标逻辑（FR-7）**

| 节点   | 逻辑/口径                                                | 单位   | 标签   | [下钻](#fr-11-下钻--二级页面规范各下钻抽屉通用)         |

## 六、交互与展示规范（沿用中心看板 UI）

- **颜色规范**：黑=正常、绿=达标、黄=接近达标、蓝=定量清零、红=重点关注、**下划线=可下钻**。
- **原则**：概览看图、细节看表；现场以表格为主，可视化为辅，需要时可导出。
- **排版**：重要数字/板块（应分拣、已分拣、待分拣、MT 号）放大突出，适当留白，提高可读性。
- **刷新**：实时刷新。
- **GOFO 设计系统**：主色 `#2f5fe0` 蓝、卡片白底浅边框 `#e8edf3`、圆角 `13-14px`、标签字体 `'Space Grotesk'`，与卸车区、概览等其他看板保持一致。
- **V2.0 静态原型约束**：移除 DC Runtime / React UMD / SC 自定义元素，二级抽屉与三级 lightbox 使用原生 `display` 属性切换；小时/累计切换、视图切换、分类标签切换控件标注"二期"占位（视觉占位、不可交互）。
